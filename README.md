# Telegram Business Monitor

Бот отслеживает редактирование и удаление сообщений (включая медиа) в ваших
личных чатах через официальную функцию Telegram **"Автоматизация чатов"**
(Telegram Business → Chatbots) и присылает уведомления прямо в чат с самим
ботом — так же, как на ваших скриншотах.

## Как это работает технически

Когда вы подключаете бота через "Автоматизацию чатов", Telegram начинает
присылать ему специальные апдейты Bot API:

- `business_connection` — статус подключения, включает `user_chat_id`
  (id вашего личного чата с ботом — туда и будут приходить уведомления)
- `business_message` — новое сообщение в любом из ваших чатов
- `edited_business_message` — сообщение отредактировано
- `deleted_business_messages` — сообщение(я) удалено

Никакого входа под вашим номером телефона не требуется — это официальный
и поддерживаемый Telegram способ, работающий через обычного бота.

## Требования

- **Telegram Premium** на вашем личном аккаунте (функция Business доступна
  только с подпиской)
- Node.js 18 или новее
- Бот, созданный через [@BotFather](https://t.me/BotFather)

## Установка

1. Установите зависимости:
   ```bash
   npm install
   ```

2. Скопируйте `.env.example` в `.env` и впишите токен бота:
   ```bash
   cp .env.example .env
   ```

3. Запустите бота:
   ```bash
   node index.js
   ```

4. В приложении Telegram (на телефоне):
   - Откройте **Settings → Telegram Business → Chatbots**
   - Нажмите "Подключить бота" (это и есть экран "Автоматизация чатов"
     с вашего первого скриншота) и выберите своего бота
   - В настройках подключения:
     - оставьте включённым доступ ко всем нужным чатам (или настройте
       исключения в "Excluded chats", если какие-то чаты не нужно отслеживать)
     - права на отправку сообщений от вашего имени можно не давать —
       боту нужно только читать сообщения

5. После подключения бот должен быть **уже запущен**, чтобы получить
   событие `business_connection` и узнать `user_chat_id`. Если вы
   подключили бота до первого запуска скрипта — просто отключите и снова
   включите бота в настройках Business, чтобы апдейт пришёл повторно.

6. Откройте личный чат с вашим ботом в Telegram — туда будут приходить
   все уведомления.

## Формат уведомлений

**Редактирование:**
```
✏️ Отредактированное сообщение
Было:
«старый текст»

Стало:
«новый текст»

От: @username (id: 123456)
Время отправки: 30.06.2026, 19:31:00
```

**Удаление:**
```
🗑 Удалённое сообщение
Содержимое:
«текст сообщения»

От: @username (id: 123456)
Время отправки: 30.06.2026, 19:31:00
```

Если в сообщении было медиа (фото, видео, голосовое, документ, стикер,
кружок, гифка) — оно пересылается отдельным сообщением сразу следом.

## Хранение данных

- `cache.json` — последние известные версии сообщений (текст + file_id медиа),
  нужен чтобы было что показать при удалении/правке
- `connections.json` — список ваших Business-подключений
- `offset.txt` — позиция в очереди апдейтов Telegram (чтобы не терять
  и не дублировать апдейты при перезапуске)

Кэш ограничен 8000 последними сообщениями, старые автоматически вытесняются.

## Важные ограничения

- Если сообщение удалили быстрее, чем бот успел получить апдейт о его
  создании, восстановить его нечем — в кэше его не будет.
- file_id медиафайлов привязан к конкретному боту и токену — он будет
  работать всё время, пока вы не пересоздадите бота.
- Если бот был выключен (не запущен) в момент удаления/правки сообщения —
  Telegram придержит апдейты и доставит их при следующем запуске
  (`getUpdates` отдаёт накопленную очередь), но не вечно — лучше держать
  бота запущенным постоянно (pm2 / systemd / screen на сервере).
- Используйте только для своего аккаунта или с согласия людей, чьи
  переписки вы отслеживаете — это могут касаться требования законов или
  правил Telegram в зависимости от вашей юрисдикции.
